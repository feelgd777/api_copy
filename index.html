Holidays

Gathers information about holidays for a specific country, province, and year. It’s perfect if you want to show holiday dates in your app, website, or service—whether for planning vacations, setting reminders, or managing events.

Parameters

Required:
  • country (string): The ISO 3166-1 alpha-2 country code (e.g., "US")
    list of country codes
  • year (string): The year for which the holidays are requested (e.g., "2023")
  • key (string): Your API key
Optional:
  • month (string): Specify the month (1-12)
  • day (string): Specify the day (1-31)
  • isPublic (boolean): Filter public holidays only (true or false)
  • subdivisions (boolean): Show holidays for subdivisions (true or false)
  • language (string): Language code (e.g., "en", "fr")
    list of language codes
  • search (string): Holiday name
  • format (string): Format (e.g., "json", "csv")

Return Properties

  • status: Status code of the API request (e.g., 200 for success).
  • totalCount: The total number of holidays returned by the API.
  • country: The ISO 3166-1 alpha-2 code. Example: "US"
  • year: The year for which the holidays are being returned.
    • name: The name of the holiday (e.g., "New Year").
    • description: A description of the holiday.
    • date: The official date of the holiday.
    • observed: The date when the holiday is observed (e.g., if the holiday falls on a weekend, it may be observed on a different date).
    • isPublic: A value of True means it is a public holiday.
      • code: The ISO code. Example: "US"
      • name: The full name of the country.
      • flag: A URL link to the country's flag.
      • date (Holiday Date)
        • name: Example: Monday
        • numeric: Example: 1 for Monday.
      • observed (Observed Date)
        • name: The name of the weekday on which the holiday is observed. Example: Monday
        • numeric: Example: 1 for Monday.

Countries

This endpoint retrieves a list of countries supported by the API. You can use the country codes from the response to query Holidays endpoint. 
Sample code will appear here...

Parameters

Required:
  • key (string): Your API key
Optional:
  • language (string): Language code (e.g., "en", "fr")
  • page (int): The page number to retrieve (useful for pagination)
  • pageSize (int): Number of records per page

Return Properties

  • totalCount: The total number of countries available.
  • page: The current page number.
  • pageSize: The number of countries returned per page.
  • totalPages: The total number of pages available.
    • code: ISO country code (e.g., "US").
    • name: The full name of the country (e.g., "United States").
    • weekend: Days of the weekend (e.g., "Sunday").
      • alpha_2: Two-letter ISO code.
      • alpha_3: Three-letter ISO code.
      • numeric: Numeric ISO code.
    • languages: A list of official languages spoken in the country.
    • currencies: The currency used in the country (e.g., "USD").
    • flag: A URL pointing to the country’s flag.
    • subdivisions: Information about the country’s states and provinces.

Languages

This endpoint retrieves a list of languages supported by the API. You can use the language codes from the response to query other endpoints like Holidays or Countries. 
Sample code will appear here...

Parameters

Required:
  • key (string): Your API key
Optional:
  • page (int): The page number to retrieve (for pagination)
  • pageSize (int): Number of records per page

Return Properties

  • totalCount: The total number of languages available.
  • page: The current page number.
  • pageSize: The number of languages returned per page.
  • totalPages: The total number of pages available.
    • code: Language code (e.g., "en").
    • name: Full name of the language (e.g., "English").
